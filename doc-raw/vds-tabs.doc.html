Tabs

Underline-first, token-driven tab navigation for dashboards, settings, and application pages. Horizontal and vertical orientations, pill and segmented variants, scrollable lists, and accessibility hooks, with no JavaScript assumptions.

Source: tabs.css · Depends on primitives.css, base.css, typography.css, utilities.css, theme tokens in /themes/*.css.

Concept & anatomy

Tabs partition content within a single page, providing local navigation between related views. VDS tabs are neutral-surface, underline-first, keyboard accessible, and fully tokenised.

Use tabs for switching between related views of the same entity (e.g. overview, activity, settings), not for primary app navigation.

  • .tabs — root container controlling orientation and density.
  • .tab-list — flex row/column with the underline baseline border.
  • .tab — interactive element (button/link) representing each tab.
  • .tab--active — selected tab with accent underline indicator.
  • .tab-panel — content region for a single tab.
HTML 
<div class="tabs">
    <div class="tab-list" role="tablist" aria-label="Project tabs">
        <button
            class="tab tab--active"
            type="button"
            role="tab"
            aria-selected="true"
            id="tab-overview">
            Overview
        </button>

        <button
            class="tab"
            type="button"
            role="tab"
            aria-selected="false"
            id="tab-activity">
            Activity
        </button>

        <button
            class="tab"
            type="button"
            role="tab"
            aria-selected="false"
            id="tab-settings">
            Settings
        </button>
    </div>

    <div
        class="tab-panel"
        role="tabpanel"
        aria-labelledby="tab-overview">
        <p>This is the Overview panel.</p>
    </div>
</div>

Horizontal tabs (default underline)

Default configuration: horizontal tab list with an accent underline on the active tab. Works on any neutral surface, inside sections, cards, or panels.

The active tab is emphasised using an underline drawn via box-shadow, aligned with the tab-list bottom border.

JavaScript only toggles .tab--active on the correct tab and shows/hides the corresponding .tab-panel.

<div class="tabs">

Density & stretch

Tabs respect the same density conventions as forms and other components. Use .tabs--a for relaxed layouts, .tabs--c for compact controls, and .tabs--stretch to make each tab share the available width.

Relaxed (A)

More generous click targets and larger text for wide layouts or primary sections.

Compact (C)

Tighter spacing for dense data views or table toolbars.

Stretch

Each tab flexes to share width, useful for segmented-like switches or pricing windows.

HTML 
<div class="tabs tabs--a">
    <div class="tab-list" role="tablist">
        <button class="tab tab--active" type="button" role="tab">General</button>
        <button class="tab" type="button" role="tab">Members</button>
        <button class="tab" type="button" role="tab">Security</button>
    </div>
    <div class="tab-panel" role="tabpanel">...</div>
</div>

<div class="tabs tabs--c">...</div>

<div class="tabs tabs--stretch tabs--c">
    <div class="tab-list" role="tablist">
        <button class="tab tab--active" type="button" role="tab">Monthly</button>
        <button class="tab" type="button" role="tab">Quarterly</button>
        <button class="tab" type="button" role="tab">Yearly</button>
    </div>
    <div class="tab-panel" role="tabpanel">...</div>
</div>

Pill & segmented tabs

Use pill tabs for filter-like groups and segmented tabs for compact view switches. Both variants drop the underline and rely on surface fills for active state clarity.

Pills read like filters and work well for short labels with optional counts.

Segmented tabs are ideal for compact mode switches inside toolbars.

HTML 
<div class="tabs tabs--pills tabs--c">
    <div class="tab-list" role="tablist">
        <button class="tab tab--active" type="button" role="tab">
            All <span class="tab__badge">12</span>
        </button>
        <button class="tab" type="button" role="tab">Active</button>
        <button class="tab" type="button" role="tab">Paused</button>
    </div>
    <div class="tab-panel" role="tabpanel">...</div>
</div>

<div class="tabs tabs--segmented tabs--c">
    <div class="tab-list" role="tablist">
        <button class="tab tab--active" type="button" role="tab">Board</button>
        <button class="tab" type="button" role="tab">List</button>
        <button class="tab" type="button" role="tab">Timeline</button>
    </div>
    <div class="tab-panel" role="tabpanel">...</div>
</div>

Scrollable tab lists

When a tab list must fit long labels or many items, opt into horizontal scrolling with .tab-list--scroll to keep the underline layout intact.

Scrollable lists keep tabs accessible without shrinking labels.

HTML 
<div class="tabs">

        <button class="tab tab--active" type="button" role="tab">Q1 Launch</button>
        <button class="tab" type="button" role="tab">Q2 Growth</button>
        <button class="tab" type="button" role="tab">Q3 Scale</button>

        <button class="tab" type="button" role="tab">2027 Outlook</button>
    </div>
    <div class="tab-panel" role="tabpanel">...</div>

Vertical tabs (settings layout)

Vertical tabs are suited for settings pages and account dashboards, where the tab list behaves like a lightweight sidebar. The indicator moves to the right edge of the active tab.

Profile

Vertical tabs align with the left edge and use a right-side indicator on the active tab. On small screens, they collapse back to horizontal.

Tabs with icons & badges

Tabs support inline icons and counters using .tab__icon and .tab__badge. These align with VDS icon and badge systems.

Icons and badges are optional and do not change the tab semantics. They are purely visual hints.

HTML 
<div class="tabs tabs--stretch tabs--c">
    <div class="tab-list" role="tablist" aria-label="Inbox filters">
        <button class="tab tab--active" type="button" role="tab" aria-selected="true">
            <span class="tab__icon">
                <span class="icon icon--mask icon--inbox"></span>
            </span>
            Inbox
            <span class="tab__badge">24</span>
        </button>
        <button class="tab" type="button" role="tab" aria-selected="false">
            <span class="tab__icon">
                <span class="icon icon--mask icon--star"></span>
            </span>
            Starred
            <span class="tab__badge">6</span>
        </button>
        <button class="tab" type="button" role="tab" aria-selected="false">
            <span class="tab__icon">
                <span class="icon icon--mask icon--archive"></span>
            </span>
            Archived
        </button>
    </div>
    <div class="tab-panel" role="tabpanel">
        <p class="text-sm">
            Icons and badges are optional and do not change the tab semantics. They are
            purely visual hints.
        </p>
    </div>
</div>

Icon-only tabs

Icon-only tabs are useful for dense toolbars. Use .tab--icon-only plus .tab__icon and always provide aria-label on each tab.

Icon-only tabs still use the same underline indicator and ARIA semantics.

HTML 
<div class="tabs tabs--c tabs--inline">
    <div class="tab-list" role="tablist">
        <button class="tab tab--icon-only tab--active" type="button" role="tab" aria-label="Inbox">
            <span class="tab__icon"><span class="icon icon--mask icon--inbox"></span></span>
        </button>
        <button class="tab tab--icon-only" type="button" role="tab" aria-label="Starred">
            <span class="tab__icon"><span class="icon icon--mask icon--star"></span></span>
        </button>
    </div>
    <div class="tab-panel" role="tabpanel">...</div>
</div>

Tabs inside a card (dashboard specimen)

A common SaaS pattern: tabs embedded into a card or panel header to switch between slices of data or different table views.

Revenue

Track performance across different time windows.

Tab panels can host charts, metrics, or tables. The card layout remains unchanged; tabs only swap the active content.

HTML 
<div class="card">
    <div class="card-header card-header--split">
        <div>
            <h3 class="text-sm font-semibold">Revenue</h3>
            <p class="text-xs text-soft">
                Track performance across different time windows.
            </p>
        </div>

        <div class="tabs tabs--c tabs--inline">
            <div class="tab-list" role="tablist" aria-label="Revenue window">
                <button class="tab tab--active" type="button" role="tab">7 days</button>
                <button class="tab" type="button" role="tab">30 days</button>
                <button class="tab" type="button" role="tab">12 months</button>
            </div>
        </div>
    </div>

    <div class="card__body">
        <div class="tab-panel" role="tabpanel">
            <p>Chart, metrics, or table go here.</p>
        </div>
    </div>
</div>

Class reference & ARIA hooks

Tabs are designed to be accessible with minimal ARIA attributes. JavaScript only needs to manage selected state and panel visibility.

Core classes

.tabs Root container for a tab set. Controls orientation and alignment.
.tabs--a Relaxed density (larger padding and font size).
.tabs--c Compact density for high-density UIs.
.tabs--stretch Stretches child tabs to share available width equally.
.tabs--vertical Vertical orientation: tab list as a column with right-side indicator.
.tabs--inline Keeps .tab-list at auto width instead of 100%.
.tabs--center Centers the tab list within the tabs container.
.tabs--right Right-aligns the tab list inside the tabs container.
.tabs--pills Renders pill-style tabs with border + filled active state.
.tabs--segmented Renders segmented tabs inside a shared surface.
.tabs--on-muted Adjusts active indicator for muted surfaces (stronger accent).

Elements

.tab-list Container for tabs. Renders the baseline border and handles orientation.
.tab-list--scroll Opt-in horizontal scrolling for long tab lists.
.tab Interactive element (button/link) representing each tab.
.tab--active Marks the selected tab and draws the accent underline indicator.
.tab--disabled Marks a tab as disabled. Also supported via the [disabled] attribute.
.tab--icon-only Compact icon-only tabs with square hit targets.
.tab-panel Content area for a single tab. Show/hide using [hidden] or .is-hidden.
.tab-panel--padded Adds extra top padding for panels inside cramped layouts.
.tab-panels Optional wrapper for multiple .tab-panel elements.
.tab__icon Optional icon container aligned with label text.
.tab__badge Small badge/counter aligned to the right of the tab label.

ARIA roles

  • role="tablist" on the .tab-list container.
  • role="tab" on each interactive .tab.
  • role="tabpanel" on each .tab-panel.
  • aria-selected="true|false" on each tab to reflect selection.
  • aria-labelledby on each panel referencing the corresponding tab id.
  • aria-orientation="vertical" on vertical .tab-list containers.

Keyboard behaviour

  • ArrowLeft / ArrowRight — move focus between tabs in horizontal lists.
  • ArrowUp / ArrowDown — move focus between tabs in vertical lists.
  • Home — focus the first tab in the list.
  • End — focus the last tab in the list.
  • Enter / Space — activate the focused tab.

Usage rules (Do / Don’t)

Patterns for correct use of tabs in application layouts.

Do

  • Use tabs for closely related views. Overview / Activity / Settings of the same entity.
  • Keep labels short and precise. Prefer 1–2 words; longer explanations live in the tab panel.
  • Use compact tabs for dense toolbars. Combine .tabs--c with .tabs--stretch when space is tight.
  • Respect focus-visible and ARIA semantics. Do not remove focus rings or omit aria-selected.

Don’t

  • Do not use tabs for primary navigation. For app-level navigation between routes or pages, use header navigation and sidebar patterns.
  • Avoid more than 6–7 tabs in a single list. If you need more, re-structure content or use nested navigation.
  • Do not mix tabs and dropdowns for the same set of options. Choose one pattern; switching between them confuses users.
  • Do not hide panels without updating ARIA. When a panel is hidden, ensure the active tab and aria-selected are updated.
