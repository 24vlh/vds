Overlays

Overlays, Modals & Drawers — System Overview

Token-driven overlay layer for backdrops, centered modals, mobile sheets, drawers and inline overlay panels. Designed to be layout-neutral and wired via JavaScript (or framework) for show/hide state.

Load order & dependencies

The overlay system builds on global structure, utilities and content blocks. Use this file only for overlay chrome and interactions, not for content layout.

Recommended load order:

  • 1. Tokens & base primitives.css, base.css, theme (slate.css, etc.)
  • 2. Layout & utilities layout.css, utilities.css, typography/content blocks
  • 3. Overlays overlays.css (this file): backdrops, modals, drawers, overlay panels

Core concepts

  • Backdrop Full-screen dim layer behind modals/drawers, with light & heavy variants.
  • Modal Centered panel with header/body/footer, sizes (sm–full), densities (A/B/C) and semantic states.
  • Drawer Left/right anchored panels with slide-in/out transitions and header/body/footer.
  • Inline overlay In-flow overlay panels with semantic variants (info/success/warning/danger).
  • Confirm dialog Compact, centered confirmation layer for destructive or blocking actions.

JavaScript contract (recommended)

  • Visibility Toggle backdrop--active, modal--active and drawer drawer--hidden via JS only. CSS does not manage state.
  • Scroll lock Apply/remove body.modal-open when any modal/sheet is open on mobile to prevent body scroll bleed.
  • Accessibility Wire role="dialog", aria-modal="true", focus management and Escape key handlers at JS level.
  • Multi-step flows Allow multiple active modals (level-2/3) for confirmations; close keys can be chained in data-close to dismiss related layers.
CSS
.backdrop { /* full-screen dim layer behind modals & drawers */ }
.modal    { /* centered modal container (flex, fixed) */ }
.drawer   { /* left/right slide-in panels */ }

Backdrops

Backdrop system — levels, density & activation

Full-screen background dimming for modals and drawers, with light/heavy opacity and multiple stacking levels for multi-modal flows.

Class reference

Backdrop classes define visual density and stacking only. Visibility is controlled via backdrop--active.

Core

.backdrop
Fixed, full-viewport layer. Default opacity via --overlay-backdrop-default.
.backdrop--light
Uses --overlay-backdrop-light for softer dimming.
.backdrop--heavy
Uses --overlay-backdrop-heavy for strong blocking overlays.
.backdrop--blocking
Heavy backdrop with blocking cursor, paired with non-dismissable dialogs.
.backdrop--active
Enables pointer events. Use this to toggle on/off via JS.

Stacking levels

.backdrop--level-2
Places backdrop above default level using z-index: calc(var(--z-backdrop) + 10).
.backdrop--level-3
Highest overlay backdrop layer z-index: calc(var(--z-backdrop) + 20).
CSS (excerpt)
.backdrop {
  position: fixed;
  inset: 0;
  background-color: var(--overlay-backdrop-default);
  backdrop-filter: blur(2px);
  z-index: var(--z-backdrop, 80);
  pointer-events: none;
}

.backdrop--light  { background-color: var(--overlay-backdrop-light); }
.backdrop--heavy  { background-color: var(--overlay-backdrop-heavy); }
.backdrop--blocking { background-color: var(--overlay-backdrop-heavy); }
.backdrop--active { pointer-events: auto; }

.backdrop--level-2 { z-index: calc(var(--z-backdrop, 80) + 10); }
.backdrop--level-3 { z-index: calc(var(--z-backdrop, 80) + 20); }

Modals

Modal base — structure, header, body, footer

Centered modals are built from a fixed container (.modal) and an inner panel (.modal__inner) with header, body and footer sections. Sizes, densities and semantics are added via modifiers.

Anatomy

The modal container handles centering and pointer event routing. The inner element acts as the actual panel, with a three-part chrome.

  • .modal Fixed full-screen flex container, centers .modal__inner. Pointer events disabled by default, enabled by .modal--active.
  • .modal__inner Panel surface; width constraints, radius, border, shadow, max-height and vertical overflow.
  • .modal__header Top chrome with title + close button; semantic states modify background/border.
  • .modal__body Scrollable main content area; flex:1 to fill height up to max-height.
  • .modal__footer Action row; right-aligned button set by default.

Example modal

This is a centered modal using the base density (.modal--b) and medium width. The body region scrolls independently if content overflows.

Base markup pattern

Canonical markup including backdrop, modal container, panel chrome, and a11y attributes. Use this as a starting point for all modal variants.

HTML
<div class="backdrop backdrop--active"></div>

<div class="modal modal--active" role="dialog" aria-modal="true"
     aria-labelledby="example-modal-title">

  <div class="modal__inner">

    <header class="modal__header">
      <h2 class="modal__title" id="example-modal-title">
        Example modal
      </h2>

      <button type="button"
              class="modal__close"
              aria-label="Close">
        &times;
      </button>
    </header>

    <div class="modal__body">
      <p>
        Body content. This area is scrollable and stretches to
        fill the available height.

      <div class="button-row">
        <button type="button">Cancel</button>

  </div>
</div>

Responsive

Mobile fullscreen modals & bottom sheets

On small viewports, modals can become full-screen experiences or bottom sheets, while still using the same HTML structure.

Fullscreen mobile

.modal--fullscreen-mobile makes the inner panel fill the entire viewport on screens up to 768px wide.

Mobile fullscreen

On mobile, this panel stretches to fill the screen with no radius.

CSS (excerpt)
@media (max-width: 768px) {
  .modal--fullscreen-mobile .modal__inner {
    width: 100%;
    height: 100%;
    border-radius: 0;
    max-width: none;
    max-height: none;
  }

  body.modal-open {
    overflow: hidden;
  }
}

Bottom sheet

.modal--sheet aligns the panel to the bottom edge and caps height at 80vh on mobile.

Bottom sheet

Ideal for quick pickers, filters, or small forms on mobile.

    align-items: flex-end;

    width: 100%;
    max-width: none;
    border-radius: var(--radius-lg) var(--radius-lg) 0 0;

}

Drawers

Left & right drawers with slide transitions

Drawers are anchored panels that slide from left or right, using the same structural ideas as modals but with distinct layout and transitions.

Class reference

Drawer classes define position (left/right), visibility and panel structure.

  • .drawer Base panel, fixed from top to bottom, with width --drawer-width.
  • .drawer--left Anchors drawer to left side of viewport.
  • .drawer--right Anchors drawer to right side of viewport.
  • .drawer--hidden Slides drawer fully off-screen and disables pointer events.
  • .drawer__header Top chrome; use for title and close button.
  • .drawer__body Scrollable content region (flex:1).
  • .drawer__footer Bottom actions; border-top separates from body.
CSS (excerpt)
.drawer {
  position: fixed;
  top: 0;
  bottom: 0;
  width: var(--drawer-width);
  max-width: 90vw;
  background-color: var(--color-surface);
  border-left: 1px solid var(--color-border-subtle);
  border-right: 1px solid var(--color-border-subtle);
  box-shadow: var(--shadow-3);
  z-index: var(--z-drawer, 95);
  display: flex;
  flex-direction: column;
  pointer-events: auto;
  transform: translateX(0);
  opacity: 1;
  transition: transform 0.25s ease, opacity 0.25s ease;
}

.drawer--left  { left: 0; }
.drawer--right { right: 0; }

.drawer--left.drawer--hidden {
  transform: translateX(-100%);
  opacity: 0;
  pointer-events: none;

.drawer--right.drawer--hidden {

Stacking & reduced motion

Overlays respect the global z-index scale and honour user motion preferences via prefers-reduced-motion.

  • Z-index tokens --z-backdrop, --z-modal, --z-drawer and --z-toast from the token set.
  • Modal/drawer transitions Translate + opacity transitions disabled when prefers-reduced-motion: reduce.
CSS (excerpt)
.modal--level-2 {
  z-index: calc(var(--z-modal, 90) + 10);
}
.modal--level-3 {
  z-index: calc(var(--z-modal, 90) + 20);
}

.backdrop--level-2 {
  z-index: calc(var(--z-backdrop, 80) + 10);
}
.backdrop--level-3 {
  z-index: calc(var(--z-backdrop, 80) + 20);
}

@media (prefers-reduced-motion: reduce) {
  .drawer,
  .modal__inner {
    transition: none;
  }
}

Accessibility & usage rules

Overlay systems require strict control over focus, scroll locking, backdrop semantics, ARIA attributes, and keyboard behaviour. Because VDS overlays are zero-animation by design, accessibility relies on structural correctness and predictable interaction rather than motion cues.

Do

  • Use real semantic containers: <dialog> (if enhanced) or div.modal with explicit ARIA.
  • Mark active modal states with: .modal--active and update aria-modal="true", role="dialog", and aria-labelledby.
  • Always pair modals/drawers with a backdrop: .backdrop--active must be present whenever the modal is interactive.
  • Ensure the close button is a real focusable element: <button class="modal__close"> with a proper .
  • Trap focus inside the active overlay (JS-controlled), cycling Back/Tab and releasing focus only when the modal closes.
  • Use body.modal-open to prevent background scrolling during modal/drawer use.
  • Maintain overflow behaviour: .modal__body and .drawer__body must keep overflow-y: auto and flex: 1 for consistent screen reader traversal.
  • Use state variants (.modal--success, .modal--danger, .modal--info, .modal--warning) only when content meaningfully matches the state.
  • On mobile (max-width: 768px), allow modal-sheet behaviour (.modal--sheet) and ensure it remains fully reachable via keyboard.
  • Provide accessible names for inline overlays: .overlay-inline + aria-labelledby or an internal heading.
  • Maintain stacking rules using level modifiers: .modal--level-2 / .modal--level-3, .backdrop--level-2 / .backdrop--level-3.
  • For blocking confirms, use .modal--blocking + data-blocking="true" and a .backdrop--blocking without a data-close handler.

Don’t

  • Don’t open a modal without also activating a backdrop.
  • Don’t hide the close button or make it an icon-only <div>. It must be a true button with aria-label.
  • Don’t disable focus-visible outlines — overlays require strong focus visibility.
  • Don’t rely on visual position to imply semantics; drawers must always be .drawer--left or .drawer--right explicitly.
  • Don’t allow background content to remain focusable while an overlay is open (must be JS-disabled).
  • Don’t place interactive content behind a loading overlay (.modal--loading) — loaders block interaction intentionally.
  • Don’t override token spacing in headers, footers, or density variants.
  • Don’t mix animation utilities or transitional CSS; overlays.css is explicitly zero-animation for stability and accessibility.
  • Don’t replace overlay surfaces with arbitrary shadows, blurs or filters — all surfaces must remain token-driven.
  • Don’t rearrange DOM order in split modals — sidebar and body positions are semantically meaningful and only collapse via responsive rules.
