Glyph system

VDS Icons System

Mask-based glyph icons, inline SVG support, semantic containers and loader patterns — all driven by tokens and currentColor.

Installation & system model

Icons are a shared subsystem used across buttons, tables, navigation and feedback. They rely on sizing tokens, mask-based glyphs and semantic containers for backgrounds and colour.

Dependencies

  • icons.css — base icon, mask glyphs, containers, utilities.
  • base.css — spacing, radius, motion tokens.
  • typography.css — text scale (for currentColor).
  • themes/*.css — semantic colours and surfaces.
  • Optional: buttons.css, tables.css, navigation.css for integrated examples.

Accessibility

Icons are usually decorative and should be hidden from screen readers, except when they are the only affordance in a control.

Decorative icon

For icons that simply support the text label, hide them with aria-hidden="true" and ensure the label text is present. 

<button class="button button--secondary" type="button">
    <span class="icon icon--mask icon--download icon--sm"
          aria-hidden="true"></span>
    <span>Download report</span>
</button>

Icon-only control

For icon-only buttons, give the control an accessible name via aria-label or visually hidden text. 

<button class="button button--ghost button--icon-only"
        type="button"
        aria-label="Close dialog">
    <span class="icon icon--mask icon--close icon--sm"
          aria-hidden="true"></span>
</button>

<button class="button button--ghost button--icon-only" type="button">
    <span class="icon icon--mask icon--close icon--sm"
          aria-hidden="true"></span>
    <span>Close dialog</span>
</button>

Accessibility & usage rules

Icon glyphs are mask-based, colour-agnostic and rely on currentColor. Accessibility requires clear naming for icon-only controls, consistent hiding of decorative glyphs, and full respect of reduced-motion preferences for loaders.

Do

  • Hide decorative icons using aria-hidden="true" while keeping the text label visible.
  • Provide aria-label or sr-only text for icon-only buttons, toggles or navigation items.
  • Use size utilities (icon--xxs … icon--xl) instead of inline pixel dimensions.
  • Use semantic icon-container--* for state surfaces (info/success/warning/danger/accent/muted).
  • Respect user motion settings: loader classes icon--spinning and icon--pulsing automatically disable animation under prefers-reduced-motion.
  • Use currentColor by changing the parent text colour rather than overriding icon colours directly.
  • Keep containers large enough for touch targets — particularly icon-container--lg and icon-container--xl.
  • Use rotation (icon--rotate-90, icon--rotate-180) only for directional UI, not to repurpose semantics.

Don’t

  • Don’t recolour icons using inline style="color: …"; use theme tokens on the surrounding element.
  • Don’t depend on icon shape alone to convey meaning — include text for critical states.
  • Don’t mix multiple icon--size classes on the same element.
  • Don’t rotate icons to imply different semantics (e.g. “danger” → rotated “info”).
  • Don’t use custom SVGs or altered mask-image values — the subsystem depends on the curated glyph set in icons.css.
  • Don’t override spacing or radius inside .icon-container; containers must follow tokenised sizes.
  • Don’t rely purely on colour in state containers — pair with a text label when used in status chips or actions. 
1. Decorative icons → aria-hidden="true".
2. Icon-only controls → aria-label or sr-only text.
3. Use icon--SIZE tokens, not inline pixels.
4. Use semantic containers (accent/success/info/…).
5. Motion utilities obey prefers-reduced-motion – do not override.
6. Do not override color: parent element sets currentColor.
7. Do not modify mask-image; use provided glyph classes only.
8. Pair icons with text for meaning-critical states.
