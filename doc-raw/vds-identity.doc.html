Identity tokens

Stroke, sizing and spacing are driven by identity tokens. Themes only supply colour; geometry is fully system-controlled.

Geometry tokens

  • --logo-stroke-primary — base stroke width.
  • --logo-clearspace — padding multiplier around mark.
  • --logo-min-horizontal — min height for full lockup.
  • --logo-min-mark — min size for mark-only.
  • --logo-primary-letter-spacing — primary wordmark tracking.
  • --logo-secondary-letter-spacing — secondary line tracking.
  • --logo-doc-height — compact inline doc height.

Size scale

  • --logo-size-xs — 2.5rem
  • --logo-size-sm — 3rem
  • --logo-size-md — 3.75rem
  • --logo-size-lg — 4.5rem
  • --logo-size-xl — 5.5rem
  • --logo-size-xxl — 6.5rem

KeepExec tokens

  • --logo-ke-frame-stroke — frame stroke width.
  • --logo-ke-mark-stroke — mark stroke width.
  • --logo-ke-text-primary-size — wordmark size.
  • --logo-ke-text-secondary-size — tagline size.
  • --logo-ke-primary-letter-spacing — primary tracking.
  • --logo-ke-secondary-letter-spacing — secondary tracking.

Inline vs external SVG usage

Choose inline SVG for dynamic theming and animation; external assets for static, cacheable logos in content surfaces.

Inline SVG

Inline SVG is recommended for app chrome, navigations and any surface where you need fine-grained control over strokes, fills or animation. 

<header>
    <a href="/">
        <svg class="logo logo--horizontal logo--sm logo--inline"
             xmlns="http://www.w3.org/2000/svg"
             viewBox="0 0 220 40">
        </svg>
    </a>
</header>

External SVG

External assets are preferred for content-rich pages and emails where caching and delivery via CMS or CDN matter more than theming.

Accessibility & usage rules

The identity subsystem is a strict, token-driven rendering contract. Logos must preserve geometric accuracy, minimum sizes, contrast requirements, and alt-text semantics whether embedded as inline SVG or external assets. Variants (mono, footer, inverse) must be selected based exclusively on background context.

Do

  • Provide meaningful alt text when using <img> logos (“Vlah Group”, “Vlah mark”), unless the logo is decorative (then alt="").
  • Use inline SVG for theme-driven recolouring via --color-logo-* tokens, and for dynamic stroke/fill adjustments.
  • Honour minimum geometry: --logo-min-horizontal and --logo-min-mark are required for readable rendering.
  • Maintain tokenised stroke geometry: --logo-stroke-primary, vector-effect: non-scaling-stroke, round linecaps/joins.
  • Use the correct variant for context: .logo--mono for single-colour layouts, .logo--footer for footer backgrounds, .logo--inverse for overlays/dark surfaces.
  • Keep the wordmark accessible by ensuring adequate contrast between --color-logo-ink / --color-logo-accent and the surface.
  • Use size utilities (.logo--xs → .logo--xxl) instead of hardcoded pixel sizing; width must remain auto.
  • Use clearspace utilities (.logo--pad-xs, .logo--pad-sm, .logo--pad-md, .logo--pad-lg, .logo--pad-xl) to preserve safe breathing room around the logo in dense UI surfaces.
  • When using mark-only icons for favicons, avatars, or compact UI, ensure the mark remains ≥ --logo-min-mark.

Don’t

  • Don’t modify paths, strokes, coordinates or proportions of the SVG files. Geometry is brand-locked and token-driven.
  • Don’t recolour logos using inline fill or stroke hex values; always use theme tokens (--color-logo-*) or appropriate variants.
  • Don’t use .logo--inverse on light backgrounds or .logo--footer outside footer surfaces; these variants are context-specific.
  • Don’t apply width/height distortions that break aspect ratio. Identity requires proportional scaling.
  • Don’t place logos on noisy, low-contrast, photographic or animated surfaces without verifying that both mark and wordmark remain legible.
  • Don’t remove vector-effect: non-scaling-stroke or override stroke-width — this breaks rendering on high-dpi screens.
  • Don’t inject non-brand shadows, filters, gradients or visual effects.
  • Don’t overlay text, icons or buttons inside the padding/clearspace boundary. 
1. Provide meaningful alt="" unless decorative.
2. Inline SVG for theme-driven recolouring; <img> for fixed artwork.
3. Always preserve proportional geometry and non-scaling-stroke.
4. Respect min sizes: --logo-min-horizontal / --logo-min-mark.
5. Use the correct variant (primary / mono / footer / inverse) for background context.
6. No hardcoded colours; use theme identity tokens only.
7. Size via .logo--xs → .logo--xxl, never by px dimensions.
8. Maintain clearspace via .logo--pad-xs/.logo--pad-sm/.logo--pad-md/.logo--pad-lg/.logo--pad-xl classes.
9. Do not edit SVG strokes, coordinates, or brand geometry.
