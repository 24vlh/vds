List surfaces

Inbox List

A responsive, row-based entity list built for inbox workflows: fast scan, predictable reflow, consistent actions, and optional expandable details content. Supports 4 densities (minimal / normal / rich / mega), plus flat vs card list rendering and flat vs rich actions.

Inbox-first scanning
Badges row, status icons, title, labels, glance, details chips, and a predictable actions rail.
Responsive reflow
Actions move below on smaller screens; title labels reflow to their own row; optional full-width toolbars.
Expandable details (CSS helpers)
Use .vds-inbox-row--open to reveal a details container. JS can toggle the class; docs include open-state markup.

Title labels reflow rules

On wide screens, .vds-inbox-row__labels sits inline to the right of the title. On small screens, the title becomes full-width and labels move to a new row aligned left. For predictable behaviour in docs, we show both containers using responsive helpers: .vds-inbox__hide-sm and .vds-inbox__show-sm.

Visibility helpers for sibling details
The base CSS does not define .vds-is-hidden/.vds-is-visible globally. If you already have global visibility helpers, use them. Otherwise, add these minimal helpers in your application layer: .vds-is-hidden { display:none; }, .vds-is-visible { display:block; }.
