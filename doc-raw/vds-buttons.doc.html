Buttons

Token-driven action triggers for primary, secondary, semantic, outline, ghost, link and icon-only interactions across the VDS surface.

Source: buttons.css · Depends on base.css, typography.css, theme tokens in /themes/*.css.

Icon buttons

Icon-only buttons keep toolbars compact. Use button--icon and provide aria-label for accessible names.

HTML 
<button class="button button--icon" aria-label="More options">
    <span class="button__icon icon icon--mask icon--more-vertical"></span>
</button>
<button class="button button--primary button--icon" aria-label="Continue">
    <span class="button__icon icon icon--mask icon--arrow-right"></span>
</button>
<button class="button button--outline button--icon" aria-label="Back">
    <span class="button__icon icon icon--mask icon--arrow-left"></span>
</button>
<button class="button button--ghost button--icon" aria-label="More">
    <span class="button__icon icon icon--mask icon--more-vertical"></span>
</button>
<button class="button button--subtle button--icon" aria-label="More">
    <span class="button__icon icon icon--mask icon--more-vertical"></span>
</button>
<button class="button button--danger button--icon" aria-label="Delete">
    <span class="button__icon icon icon--mask icon--more-vertical"></span>
</button>

Link buttons

Link buttons remove borders and background fill. Use them for inline or low-emphasis actions.

HTML 
<a href="#" class="button button--link">View all</a>

    <span class="button__icon button__icon--left icon icon--mask icon--arrow-right"></span>
    <span>Read more</span>
</button>
<button class="button button--link">
    <span>Next</span>
    <span class="button__icon button__icon--right icon icon--mask icon--arrow-right"></span>
</button>
<button class="button button--link" disabled>Disabled link</button>

Installation & dependencies

Buttons are a surface-level component depending on base reset, typography, tokens, and active theme colours. Include files in this exact order.

  • Load base.css and layout.css first.
  • Load typography.css and icons.css.
  • Load active theme: themes/slate.css, navy.css, carbon.css.
  • Load buttons.css as the dedicated button layer.

Anatomy

The canonical button is a flex container with optional leading/trailing icon, label, and semantic modifier classes.

HTML 
<button class="button button--primary">
    <span class="button__icon icon icon--mask icon--arrow-right"></span>
    <span>Primary action</span>
</button>

Required: button base class.   Optional: semantic modifiers (button--primary, button--secondary, etc.), size modifiers (button--xs, button--sm, button--md, button--lg), layout modifiers (button--block, button--icon).

All visual variants share the same hit-area, typography and motion tokens. Only colour and border semantics change between variants.

Sizes & layout

Size modifiers adjust padding and font-size. Block layout is the only width modifier and should be used sparingly for primary flows.

HTML 
<button class="button button--primary button--xs">Extra small</button>
<button class="button button--primary button--sm">Small</button>
<button class="button button--primary button--md">Medium</button>
<button class="button button--primary button--lg">Large</button>

<button class="button button--primary button--block">
    Block level primary
</button>

button--xs matches compact action buttons (e.g. inbox toolbars).

Add button--lift for the subtle hover raise used by inbox actions.

Icons & loading

Icon-only buttons use button--icon (alias button--icon-only). Loading state is a visual modifier that preserves layout and width.

HTML 
<button class="button button--secondary">
    <span class="button__icon button__icon--left icon icon--mask icon--arrow-left"></span>
    <span>Back</span>
</button>

<button class="button button--primary">
    <span>Continue</span>
    <span class="button__icon button__icon--right icon icon--mask icon--arrow-right"></span>
</button>

<button
    class="button button--ghost button--icon"
    type="button"
    aria-label="More options">
    <span class="button__icon icon icon--mask icon--more-vertical"></span>
</button>

<button class="button button--primary button--loading" disabled>
    <span>Saving…</span>
</button>

Accessibility & usage rules

Contract for safe, semantic, token-driven buttons. Maintain correct element semantics, provide accessible names, and rely on VDS colour, motion and focus tokens rather than ad-hoc styling.

Do

  • Use real <button> elements for actions; reserve <a> for navigation.
  • Provide aria-label for icon-only buttons (button--icon, button--icon-only).
  • Use one clear primary action per page or workflow.
  • Use size modifiers (button--xs, button--sm, button--md, button--lg) instead of custom padding.
  • Use semantic variants (--success, --warning, --danger, --info) to express meaning consistently.
  • Keep disabled state using disabled or aria-disabled="true" to ensure proper assistive behaviour.
  • Use button--loading to preserve layout width and avoid content shifts.
  • Allow tokens to determine focus ring, colours, and transitions (do not override them in components).

Don’t

  • Don’t remove focus-visible rings or replace them with non-token shadows.
  • Don’t use colour alone to indicate destructive actions: rely on button--danger or outline-danger.
  • Don’t nest block-level buttons inside inline-only containers.
  • Don’t mix manual inline styles for padding, radius or borders.
  • Don’t use button--block excessively; reserve it for primary, high-friction flows.
  • Don’t create new ad-hoc variants; use semantic or outline APIs already provided.
  • Don’t animate or spin content manually during loading; use the built-in spinner logic.
1. Use <button> for actions; provide aria-label for icon-only.
2. Use one primary action per view; avoid colour-only meaning.
3. Use size modifiers (button--xs/sm/md/lg), never inline padding.
4. Keep focus-visible ring from tokens; do not suppress it.
5. Use semantic or outline variants for meaning (danger/warning/etc.).
6. Preserve layout in loading state with .button--loading.
7. Avoid overriding token-driven colours, borders, or motion.
