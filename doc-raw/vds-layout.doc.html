Foundations

VDS Layout System

The layout layer defines the structural skeleton of VDS: page shell, containers, vertical rhythm, structural grids, and semantic layouts. It is token-driven, theme-agnostic, and purely concerned with structure.

Layout foundation

layout.css exposes a small set of structural primitives: page frame, reusable content regions, containers, section spacing, grids, and high-level layouts. All values are driven by tokens from primitives.css.

Page frame

  • header.page-header — global site header.
  • main.page-content — primary page content.
  • footer.page-footer — global footer.

Content regions

  • .content-header — local header area.
  • .content-body — flex-filling content area.
  • .content-footer — local footer/actions.

Containers & grids

  • .container* — max-width shells.
  • .section* — vertical spacing helpers.
  • .layout-grid-* — structural grids.

Page shell

The page shell is semantic-bound and single-instance: one header, one main content region, one footer. layout.css wires them into a flex column shell with a growing main region.

Accessibility & usage rules

The layout layer provides structural semantics and predictable reading order. It must remain colour-neutral, typography-neutral, and free of interactive behaviour. Accessibility is guaranteed through correct landmarks, consistent DOM order, responsive collapse rules, and token-driven spacing.

Do

  • Use semantic landmarks for the page shell: <header class="page-header" role="banner">, <main class="page-content" role="main">, <footer class="page-footer" role="contentinfo">.
  • Keep DOM order meaningful — layout utilities (split, sidebar-layout, layout-grid-*) must not be used to reorder semantics. Visual order may change; reading order must not.
  • Allow content regions (.content-header, .content-body, .content-footer) to behave as true structural sections for modals, panels, dashboards and shells.
  • Use max-width containers for readability: .container, .container-wide, .container-narrow, .container-tight, .container-relaxed.
  • Apply .section* classes to maintain consistent vertical rhythm driven by tokens (--section-spacing-*).
  • Let structural grids collapse at mobile breakpoints exactly as defined: @media (max-width: 768px) → single column.
  • In sidebar layouts (.sidebar-layout, .sidebar-layout-right), keep the sidebar as the first semantic element in the DOM, even if visually moved or collapsed.
  • Use full-bleed patterns (.full-bleed, .section-bleed) only to extend background/surfaces — content must remain inside containers for readable line lengths.
  • Ensure the document shell remains height-stable: main.page-content { flex: 1 } is required for accessible viewport fill.

Don’t

  • Don’t inject colours, typography, borders or component visuals into layout primitives — layout.css is structure-only.
  • Don’t use grid or split layouts to reassign semantic order (e.g., placing the footer before the main content visually).
  • Don’t override container max-widths with arbitrary pixel values — always use the provided token-based container classes.
  • Don’t attach interaction behaviour (toggles, overlays, JS-driven transitions) to layout classes.
  • Don’t collapse structural grids using custom breakpoints — follow the system breakpoint at 768px for grid and split collapse.
  • Don’t “double-wrap” content in competing containers (e.g., .container inside .container-tight).
  • Don’t hardcode padding or margins inside layout classes — spacing is managed exclusively via tokens and section helpers.
  • Don’t attempt to modify or remove min-height: 0 from .content-body; scrollable regions depend on this for correct overflow behaviour in flex parents. 
1. Use header/main/footer landmarks correctly; one instance each.
2. Never reorder semantic DOM structure using CSS grid/flex.
3. Content regions must remain scroll-safe: min-height: 0 is required.
4. Use containers for max-width readability; never hardcode widths.
5. Maintain section spacing using token-driven .section* helpers.
6. Structural grids collapse to 1fr at 768px — don’t override this.
7. Sidebar layouts collapse to single-column at 1024px; sidebar stays first.
8. Full-bleed patterns expand background only, not content width.
9. Layout.css must contain no colours, typography or component logic.
