Usage patterns & best practices

Description lists are best for compact metadata and attribute summaries. Keep terms short and values scannable.

Do

  • Use description lists for metadata panels and summaries.
  • Keep terms short and align values for quick scanning.
  • Use inline layout for short lists and stacked for narrow spaces.
  • Use borders or stripes to separate dense lists.
  • Align related lists with the same column count.

Do not

  • Do not use description lists for primary content or paragraphs.
  • Do not overload values with long sentences.
  • Do not mix unrelated terms in the same list.
  • Do not use inconsistent term casing or punctuation.
  • Do not nest complex interactive controls inside values.

Accessibility & usage rules

Use semantic <dl>, <dt>, and <dd> elements so assistive technologies can announce term and value pairs correctly.

Do

  • Use real <dl>, <dt>, <dd> elements.
  • Keep each term paired with its description.
  • Use clear labels for abbreviated terms.
  • Keep contrast strong for terms and values.

Do not

  • Do not replace <dt> with generic headings.
  • Do not use lists for tabular data.
  • Do not remove spacing that makes pairs unclear.
  • Do not place unrelated controls inside <dt>.
